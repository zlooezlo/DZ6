# Starter kit для домашнего задания Kubernetes

Этот архив снимает неоднозначность: backend не нужно придумывать с нуля.
Студент получает минимальное приложение и должен:

1. изучить код и объяснить endpoints;
2. собрать и опубликовать образы `v1` и `v2`;
3. развернуть backend вместе с PostgreSQL;
4. выполнить RollingUpdate и rollback;
5. доказать отсутствие ошибок во время обновления;
6. при желании доработать приложение, но сохранить `/healthz`, `/readyz` и `/api/visits`.

## Состав

- `backend/` - Python/Flask backend и Dockerfile;
- `k8s/` - базовые манифесты backend;
- `build-images.sh` - сборка и публикация v1/v2;
- `ROLLING_UPDATE.md` - пошаговая лабораторная.

## Важное ограничение

В production нельзя считать один успешный rollout доказательством zero downtime. Для домашнего задания студент должен показать непрерывный поток запросов без HTTP-ошибок и объяснить зависимость результата от readinessProbe, ресурсов, PDB, Ingress/Service и доступности БД.
