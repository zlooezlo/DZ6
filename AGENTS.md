# Правила проекта DZ6

## Коммиты

- Использовать короткие технические формулировки без префиксов Conventional Commits.
- Формулировки должны точно перечислять ключевые изменения.
- Не использовать юмор в сообщениях коммитов.
- Не объединять несвязанные изменения в один коммит.
- Не добавлять реальные пароли, токены, kubeconfig и другие секреты.

Примеры:

- `Настроены NFS CSI Driver v4.13.4 и StorageClass nfs-csi`
- `Добавлены Deployment и Service для backend`
- `Настроены RollingUpdate и rollback backend`
- `Добавлены NetworkPolicy для frontend, backend и PostgreSQL`

## Проверка изменений

- Проверять YAML до коммита.
- Проверять сборку изменённых контейнерных образов.
- Для Kubernetes-изменений сохранять команды и доказательства в `evidence/`.
- Настоящие значения Secret хранить только вне Git.
