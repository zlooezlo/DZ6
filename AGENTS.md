# Правила проекта DZ6

## Коммиты

- Использовать понятный формат Conventional Commits: `type(scope): message`.
- Формулировки должны точно описывать изменение.
- Допускается и приветствуется лёгкий, оригинальный юмор без потери технического смысла.
- Не объединять несвязанные изменения в один коммит.
- Не добавлять реальные пароли, токены, kubeconfig и другие секреты.

Примеры:

- `chore(repo): распаковали starter kit — кластер уже волнуется`
- `fix(frontend): вернули стили, потому что голый HTML замёрз`
- `feat(postgres): прикрутили PVC, теперь данные не кочевники`
- `test(rollout): v2 пришла без остановки оркестра`

## Проверка изменений

- Проверять YAML до коммита.
- Проверять сборку изменённых контейнерных образов.
- Для Kubernetes-изменений сохранять команды и доказательства в `evidence/`.
- Настоящие значения Secret хранить только вне Git.
